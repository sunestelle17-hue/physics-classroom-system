# 物理课堂登记系统原型

一个面向手机竖屏课堂快速点击场景的物理课堂登记网页系统第一版原型。当前版本只使用浏览器本地模拟数据与 `localStorage`，不连接真实数据库，也不录入真实学生姓名。

## 已实现功能

- 选择年级、班级、讲次、模块和学生。
- 按“一个学生一组题”的方式登记题目表现。
- 每道题支持登记：
  - `√`：独立正确
  - `△`：提示后正确
  - `×`：错误或不会
  - `—`：已布置但未完成或缺席
  - `空白`：尚未检查
- 支持整道题或整个模块标记为“未布置”，未布置内容不计入完成进度。
- 当题目标记为 `△` 或 `×` 时，可以多选错因，错因不是必填。
- 支持课堂表现标签和一行自由备注。
- 点击和输入会自动保存到浏览器 `localStorage`。
- 切换学生后再切回来，之前登记的数据仍会保留。
- 展示当前学生完成进度，例如 `6/10`。
- 内置虚拟年级、虚拟班级、虚拟学生和虚拟题目，方便演示。
- 手机竖屏优先，按钮尺寸较大；桌面端也可正常使用。
- 管理入口支持新增、修改、删除年级、班级和学生。
- 新增学生时可选择所属年级和班级，并支持每行一个姓名的批量粘贴。
- 支持本地 JSON 导出和导入恢复。
- 删除学生、班级、年级以及清空演示数据前会二次确认。
- 可清空内置演示数据；真实姓名仅保存在浏览器 localStorage 或用户自行导出的 JSON 文件中。
- 可根据题目状态、错因、课堂表现标签和自由备注生成可编辑的中文家长反馈草稿，并支持一键复制；系统不会自动发送。

## 技术方案

- 零构建静态网页
- 原生 JavaScript
- 原生 CSS
- 浏览器 localStorage

## 本地运行

启动开发服务器：

```bash
npm run dev
```

运行语法检查：

```bash
npm run build
```

预览静态页面：

```bash
npm run preview
```

## 部署

本项目已配置 GitHub Pages 自动部署。每次推送到 `main`、`master` 或当前 `work` 分支后，GitHub Actions 会运行 `.github/workflows/pages.yml`，构建 `dist` 静态产物并发布到 GitHub Pages。

构建产物使用相对路径引用资源，因此既可以部署到 GitHub Pages 用户站点，也可以部署到项目站点子路径。

## 当前暂未完成

第一版原型刻意不包含以下能力：

- 真实数据库、登录和权限。
- 真实学生姓名录入。
- 题库、答案、OCR、自动组卷。
- 复杂统计分析和导出报表。
- 多设备实时同步。
